# 2026-05-18
<!-- DAILY_CHECKIN_2026-05-18_START -->
今天是 AI x Web3 学习 Day 1，我参加了残酷共学和分享会，重点学习了 Web3 系统设计、钱包、交易生命周期和服务端钱包安全。

今天最大的收获是：Web3 不是简单地把支付系统换成链上版本，而是把传统支付里的部分信任角色，从银行/支付机构转移到区块链、智能合约和密码学签名上。钱包也不只是资产入口，而是身份、私钥和签名授权的关键界面。

我对交易生命周期的理解更清楚了：一笔交易通常会经历构造、签名、广播和确认，关键参数包括 Gas、Nonce 和 Data，服务端还需要通过交易确认和 Event 监听来判断业务状态。今天特别记住了“交易模拟”的重要性，它可以在真正广播前帮助用户理解交易后果。

结合 AI x Web3，我现在更关注安全边界：AI 可以帮助用户解释交易、整理信息、做风险提示和模拟结果，但签名、授权和资产相关动作必须保留明确的人类确认。

我还不清楚的问题是：一个面向普通用户的 AI x Web3 产品，应该如何把签名风险讲清楚，同时又不让用户被复杂术语吓退？

明天我计划继续学习 LLM / Prompt / Context，把今天的 Web3 系统设计内容和 AI 辅助用户教育的场景连接起来。
