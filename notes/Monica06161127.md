**今日学习主题：AI Native × Web3 Native 范式 + 开发者工具链**  
  
一、AI Native（AI 原生）  
\- 定义：从设计之初就把大模型作为核心底层组件，去掉 AI 就无法运行  
\- 对比：AI 赋能 = 传统产品 + AI 插件（去掉还能跑）；AI Native = AI 是心脏（去掉就死）  
\- 四大特征：模型底层嵌入、自然交互主导、数据闭环迭代、不可移除性  
\- 例子：ChatGPT、Perplexity、Cursor  
  
二、Web3 Native（Web3 原生）  
\- 定义：从设计之初就把区块链、去中心化、Token 经济作为 DNA  
\- 对比：Web2.5 = 传统产品 + 区块链插件；Web3 Native = 区块链是 DNA（去掉就不是它了）  
\- 六大特征：去中心化架构、用户真实所有权（Read+Write+Own）、Token 原生经济、信任最小化、可组合性（货币乐高）、无需许可  
\- 例子：Uniswap、Aave、Decentraland  
\- 趋势：AI Native × Web3 Native 正在融合（AI Agent + 链上执行）  
  
三、三大协议：Agent 如何连接世界  
\- MCP（Model Context Protocol）：连接大模型与外部数据源/工具的统一标准协议，让 AI 能"看到"外部世界  
\- ACP（Agent Client Protocol）：规范代码编辑器与 AI 编码助手之间的通信协议，让 AI 能"操作"编辑器  
\- Function Call：LLM 调用外部函数/工具的底层机制，MCP/ACP 建立在其之上  
\- 关系：Function Call 是神经系统，MCP 是五官（感知），ACP 是四肢（操作）  
  
四、开发者工具链（GitHub 生态）  
\- Issue：问题记录本 + 任务看板（Bug 报告、功能请求、任务分配）  
\- Pull Request：代码变更的正式合并申请（本地修改 → git push → 创建 PR → 审查 → 合并）  
\- Release：正式版本发布（Git Tag + 二进制文件 + Changelog）  
\- GitHub CLI（gh）：终端里操作 GitHub 的命令行工具，常用 gh pr / gh issue / gh repo  
\- HTTPS vs SSH：HTTPS 用 Token 认证（简单），SSH 用密钥对认证（配置一次后免密）  
\- Gist：轻量级代码片段分享（代码界的 Pastebin）  
\- Codespaces：云端完整 VS Code 开发环境（按小时计费）  
  
五、编程基础  
\- Typeless（无类型语言）：没有类型系统的语言（如 BCPL、汇编），对比动态类型语言（Python/JS 有运行时类型）  
  
六、核心收获  
\- "不可移除性"是判断 Native vs Enabled 的黄金标准：去掉这个技术，产品还能活吗？能活 → Enabled；不能活 → Native  
\- MCP + ACP = AI Agent 的"五官和四肢"，Function Call 是神经系统  
\- GitHub 工具链 = Issue → PR → Release 的完整闭环
