记录今天的三个模块👇

**模块一：AI Agent 的「记忆」问题**

**核心洞察：AI的终极形态不是更聪明的聊天机器人，而是能“记住你”的长期伙伴。**

**为什么AI需要记忆？**  
比如自己每天去同一家咖啡店，店员每次都问“请问您要喝什么？”——自己每天都点的“少糖燕麦拿铁”，他完全不记得（我简单联想一下哈）。这就是之前的ChatGPT。每次开启新对话，个人的偏好、工作进展以及决策等等全部清零。这不仅会耗费精力重新和AI对话，还会降低交流的欲望（想想就心累😩）。_所有AI Agent能从“一次性工具”转变成“长期伙伴”，其核心就在于Memory。_

**记忆比想象中难**  
设计的出发点是基于人脑，不是简单把对话历史存下来就完事了。会划分为短期，长期等。且一条记忆会关于以下三点：第一，记忆板块权重变化。第二，记忆会有冲突——比如第一天说“简洁表述”，隔了一天又要求“写详细点”，AI该选择听哪个？第三，检索比存储更难——学习也是这样，不断的输入，积累所习得的知识，应用时有的放矢的输出，输出往往比输入更重要。_还有一个思考的点：所有权问题。AI记忆到底属于自己还是平台，换了一个AI助手，所承载的记忆能带走吗？同时，我又想到了蒸馏skill，这两个感觉还蛮像的。一个是从内部出发，一个是从外部“炼化“🤔。_

**Coding Agent是最需要记忆的场景**  
就像自己在开发一个持续几周的软件项目。如果AI不记得你的技术栈、之前做了哪些决策以及哪些方案试过失败了，那么每次都要花大量时间和他讲背景信息。现在可以用md文件给AI写“项目记忆手册”，比如Claude.md，可以自己定人设🎭。那么未来的方向可能是AI自己维护和更新这份记忆（不确定现在进化到什么程度💭，写完打卡笔记马上去查）。

**模块二：Web3 的「隐私」悖论**

**区块链上的每一笔交易全世界都能看到。**

**区块链天生透明，不是匿名**  
跟谁交互、转了多少钱、什么时候转的，全都公开可查。🥔老师说这是“裸奔”hhh。

**钱包可能“出卖”你**  
MetaMask发交易时，发出者的IP地址会被RPC节点记录。而很多RPC都架设在AWS上，相当于你的身份信息被中心化服务器存着。\_去中心化的承诺，败给了中心化的基础设施。\_而且自己甚至无法验证他们是否真的不会泄露数据，相当于还是需要**信任**机制。

**隐私协议怎么做到"又交易又隐藏"？**  
Railgun用的是UTXO模型——资金像持有的多张独立钞票，而不是银行卡余额的一个数字。每张钞票互不干扰，可以在隐私状态下去DeFi里操作。同时还有一个“清白证明”系统（POI）：你的钱进隐私池后，系统自动检查来源是否干净，发一个零知识证明给你。有了它，交易所不会冻你的账户——既保护隐私，又合规。

**一个反直觉的结论**  
越多人用隐私协议，隐私性反而越好。池子里人越多，越难分辨是谁的交易——“大海捞针”效应。

**模块三：金融生的「生态位」在哪？**

**金融生在Web3领域的不可替代性或许在于\_看懂数学+会用数据+能设计Tokenomics\_。**

**学校课程的滞后性**  
国内大一金融课程偏宏观经济、货币银行学等理论，教材更新慢。而实际的国际金融业界更看重公司金融、数据分析和实战案例。

**找到交叉点**  
跟AI聊了一圈职业方向，发现还是不要同时学Python+Excel+托福+研报+原版教材，信息过载真的会压垮自己😭（5.21已经反思过，二八定律）。正确的做法是找到金融+AI+Web3的交叉生态位——用金融逻辑审视Web3项目（Tokenomics设计、财务分析），用AI工具提效（数据处理、研报生成）。

**“输出驱动”比“输入驱动”重要**  
写一篇项目研报比听十节课有用。_真正的理解来自输出——把自己学到的东西写出来、讲出来、做出来_。这也是为什么大家会在共学营中打卡记录。

今天最大的感受：这三个话题看似独立，其实都在回答同一个问题——**在透明的世界里，如何保护个体的边界？**

AI记忆关乎“你的数据归谁”，链上隐私关乎“你的交易信息归谁”，职业规划关乎“你的过程和成果归谁”。时间在变，技术在变，但底层问题一直没变。

明天继续，共勉～💪

Web2到Web3的演变，本质上是信任机制的迁移。传统支付是用户→银行→央行→银行→商户，我们相信的是机构背书。Web3是用户→钱包→区块链→智能合约→商户，相信的是数学和代码。这听起来很理想，但前提是代码真的没问题。

今天学到很多概念：区块链是全民记账的账本，智能合约是自动执行的代码合同，共识机制决定了全网怎么达成一致（PoW比算力、PoS比质押、DPoS比投票、PoH比时间）。交易从构造到确认要经历签名、广播、Mempool、矿工打包这些环节，中间还有个"交易模拟"的安全检查站。

钱包体系也很有意思：EOA是最基础的，合约钱包和AA钱包（ERC-4337）更智能，MPC钱包用多方计算提升安全性，多签钱包适合团队场景。托管/非托管的选择，本质上是便利性和安全性的权衡。安全方面，老师反复强调Security by Design。Approve授权的风险、EIP-712的可视化签名、TEE可信执行环境。

**今日学习主题：AI Native × Web3 Native 范式 + 开发者工具链**  
  
一、AI Native（AI 原生）  
\- 定义：从设计之初就把大模型作为核心底层组件，去掉 AI 就无法运行  
\- 对比：AI 赋能 = 传统产品 + AI 插件（去掉还能跑）；AI Native = AI 是心脏（去掉就死）  
\- 四大特征：模型底层嵌入、自然交互主导、数据闭环迭代、不可移除性  
\- 例子：ChatGPT、Perplexity、Cursor  
  
二、Web3 Native（Web3 原生）  
\- 定义：从设计之初就把区块链、去中心化、Token 经济作为 DNA  
\- 对比：Web2.5 = 传统产品 + 区块链插件；Web3 Native = 区块链是 DNA（去掉就不是它了）  
\- 六大特征：去中心化架构、用户真实所有权（Read+Write+Own）、Token 原生经济、信任最小化、可组合性（货币乐高）、无需许可  
\- 例子：Uniswap、Aave、Decentraland  
\- 趋势：AI Native × Web3 Native 正在融合（AI Agent + 链上执行）  
  
三、三大协议：Agent 如何连接世界  
\- MCP（Model Context Protocol）：连接大模型与外部数据源/工具的统一标准协议，让 AI 能"看到"外部世界  
\- ACP（Agent Client Protocol）：规范代码编辑器与 AI 编码助手之间的通信协议，让 AI 能"操作"编辑器  
\- Function Call：LLM 调用外部函数/工具的底层机制，MCP/ACP 建立在其之上  
\- 关系：Function Call 是神经系统，MCP 是五官（感知），ACP 是四肢（操作）  
  
四、开发者工具链（GitHub 生态）  
\- Issue：问题记录本 + 任务看板（Bug 报告、功能请求、任务分配）  
\- Pull Request：代码变更的正式合并申请（本地修改 → git push → 创建 PR → 审查 → 合并）  
\- Release：正式版本发布（Git Tag + 二进制文件 + Changelog）  
\- GitHub CLI（gh）：终端里操作 GitHub 的命令行工具，常用 gh pr / gh issue / gh repo  
\- HTTPS vs SSH：HTTPS 用 Token 认证（简单），SSH 用密钥对认证（配置一次后免密）  
\- Gist：轻量级代码片段分享（代码界的 Pastebin）  
\- Codespaces：云端完整 VS Code 开发环境（按小时计费）  
  
五、编程基础  
\- Typeless（无类型语言）：没有类型系统的语言（如 BCPL、汇编），对比动态类型语言（Python/JS 有运行时类型）  
  
六、核心收获  
\- "不可移除性"是判断 Native vs Enabled 的黄金标准：去掉这个技术，产品还能活吗？能活 → Enabled；不能活 → Native  
\- MCP + ACP = AI Agent 的"五官和四肢"，Function Call 是神经系统  
\- GitHub 工具链 = Issue → PR → Release 的完整闭环
