思考錢包和ai交易的交叉

Human-in-the-loop。  
AI 可以準備交易，但最後由人確認。

常見場景：

-   第一次付款給新地址
    
-   超過限額
    
-   跨鏈轉帳
    
-   授權智能合約
    
-   涉及高價資產
    
-   AI 判斷信心不足
    

**子錢包機制**  
不要讓 AI 使用主錢包，而是給它一個獨立子錢包。

好處：

-   只放少量資金
    
-   降低主資產風險
    
-   容易停用
    
-   方便追蹤 AI 的支出
    
-   不同 AI Agent 可使用不同錢包

# 今天將Hemers安裝到外接硬盤上

首先在外接硬盤上裝上WSL

然後配置環境

```
wsl -d Hermes-Ubuntu-22.04
```

安裝並驗證

```
hermes --version
```

並安裝好Node v22.22.3、npm 10.9.8、ripgrep、ffmpeg，把 hermes 固定到 /usr/local/bin/hermes。我也關掉了這個 WSL 的 Windows PATH 繼承，避免 Windows 路徑裡的括號干擾 Linux 指令。  
  
下一步配置API Key  

```
wsl -d Hermes-Ubuntu-22.04
```

然後在WSL裡  

```
hermes setup
```

最後啟動

```
hermes
```
