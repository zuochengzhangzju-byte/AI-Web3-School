今天主要學習 memory 透過演講

memory 有是個蓋擴 內容很廣

內容分

### Prompt Engineering（這次怎麼問）

就是你每次送給 Claude 的那個 request 本身。包含 system prompt、使用者訊息、工具定義。它只管「這一次」的請求要怎麼組。

記憶可以被**引用**在這層（例如 system prompt 裡寫「使用者偏好 pnpm」），但 prompt 本身不負責決定什麼記憶該存、該刪。

### Context Engineering（什麼東西進 window）

在送出 request 之前，決定 context window 裡要放什麼。這就是我上一則講的 Level 0 到 Level 3 的差別：全塞歷史？只塞最近 50 條？還是用 RAG 搜出相關片段再塞？

記憶**主要住在這層**。因為「從持久化的記憶裡撈出相關的東西，組裝進 context window」這個動作就是 context engineering。

### Harness Engineering（系統怎麼跑）

Agent 的執行框架：tool call 的迴圈、錯誤處理、日誌紀錄、權限控管、生命週期管理。例如 Claude Code 的 agent loop——Claude 呼叫工具、拿到結果、決定下一步、再呼叫工具——這整個迴圈的管理就是 harness。

記憶在這層的角色是**可觀測性和控制**：什麼時候觸發記憶寫入、什麼時候清理過期記憶、寫入失敗怎麼處理。
