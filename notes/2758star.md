# 2026-05-22
<!-- DAILY_CHECKIN_2026-05-22_START -->
今天实时参加了例会和co-learning，整体感受是这场例会不只是简单的信息同步，更像是一次学习方法和实践经验的集中分享。会上多位同学分享了自己在 AI、Agent、Web3 学习过程中的工具使用、踩坑经验和个人方法论，让我对后续如何参与课程、完成作业和沉淀学习成果有了更清晰的认识。

## 一、Week 1 例会收获

本次例会中，我印象比较深的是几位同学围绕 Agent、学习自动化、知识管理和技术实践做了分享。

首先，有同学分享了自己在调用课程平台 API 时遇到的坑。例如某些任务接口如果不传入具体 task id，并不会默认返回所有任务，而是返回空数组。因此他通过先获取全部 task id，再循环调用 history 接口的方式，拼接出学习进度报告，并进一步沉淀成脚本，用来辅助打卡、同步课程大纲和生成学习材料。这个分享让我意识到，Agent 和 API 不只是课程里的理论内容，也可以反过来服务自己的学习流程，把学习、记录、提交、复盘变成一个自动化 workflow。

其次，另一位同学分享了使用 Obsidian + Agent 搭建个人知识库的过程。她提到，新手不需要一开始就设计特别复杂的文件夹结构，而是可以先保持一到两层的浅层分类，让系统先跑起来。她还分享了 MOC、PARA、Daily Note、Templator 等方法和插件，用来管理学习笔记、概念笔记和日常反思。对我启发最大的一点是：AI 可以帮助整理资料、生成结构和辅助归档，但真正的反思内容仍然需要自己完成。因为如果连 reflection 都完全交给 AI，久而久之可能会分不清哪些想法是真正来自自己。

此外，也有技术背景的同学提醒大家不要被层出不穷的新工具、新概念带来焦虑。很多新工具本质上是在类似的底层框架上增加新的模块或功能，与其每天追逐新名词，不如理解它们背后的核心结构和工作逻辑。这让我对后续学习 AI/Web3 有了更稳定的心态：不需要一看到新工具就慌张，而是要先判断它解决了什么问题、和已有工具有什么关系、是否真的适合自己的学习或项目需求。

## 二、Co-learning 环节收获：了解作业提交规范

在 Co-learning 环节中，我重点了解了作业提交规范。相比单纯完成任务，我意识到更重要的是形成一个规范的学习闭环：学习之后要及时整理内容，按照要求提交作业，并保留自己的学习证明和过程记录。

这让我明确了后续参与课程时需要注意几点：

1.  每次学习后要及时整理当天内容，而不是等到最后再补。
    
2.  作业提交不仅是完成任务，也是对自己学习过程的记录。
    
3.  提交内容应尽量包含自己的理解、问题和下一步行动，而不是只复制课程内容。
    
4.  可以借助 AI 帮助整理结构，但核心观点和反思应该由自己完成。
    
5.  要关注平台和社群中的最新通知，避免错过课程安排、提交要求或时间变动。
    

## 三、我的思考

今天最大的收获是：学习 AI/Web3 不能只停留在听课和看资料层面，而要尽快建立自己的学习系统。这个系统可以包括笔记工具、GitHub、Agent、自动化脚本和每日复盘。工具的作用不是替代自己学习，而是帮助自己更好地记录、整理和迭代。

同时，我也意识到自己作为初学者，不需要一开始就追求复杂的技术实现。更实际的做法是先把每天的学习内容记录下来，按照作业规范完成提交，再逐步尝试用 AI 工具优化自己的学习流程。

## 四、下一步行动

接下来，我会从两个方面开始行动：

第一，按照 Co-learning 环节了解到的作业提交规范，认真完成每天的学习记录和打卡，保证内容中有自己的理解和反思。

第二，尝试搭建一个简单的个人学习笔记系统，用来记录课程内容、关键概念、问题清单和每日复盘。前期不追求复杂分类，先保持简单、可持续，再根据学习进度慢慢优化。

# 2026-05-19

上午推了两个 WCB 任务。一个是测试网交易，在 Sepolia 用 MetaMask 发了笔 0.00001 ETH 到销毁地址，Gas 21000，烧了大概 0.000031 SepoliaETH——第一次盯着 Etherscan 看 Gas 怎么算，感觉终于不是走过场了。另一个是做了个叫「签还是不签」的 Web 工具，模拟五个合约授权场景，接 DeepSeek API 实时分析风险。做这个是因为我发现很多新手（包括之前的我）看到钱包弹窗就直接点 confirm，根本不看提示，想训练一下"签名前停三秒"的意识。累计 17/37，305pt。

今天的 Prompt 学习最有收获的一句话是：Prompt 是软约束不是安全边界。学 Prompt Injection 的时候才真正理解这件事。一句"忽略之前所有规则"就能让模型切换模式，system prompt 根本挡不住，真正的防护得在应用层做。Few-shot 我之前理解也是错的，以为是在教模型新知识，其实只是给格式锚点。下午实时参加了 Hermes Agent 入门分享，工具调用、记忆、任务分解这几块串起来了，Agent 不再是魔法了。

项目拆解读了 Bittensor 和 Render Network。Bittensor 我花时间最多，直接进官方文档看子网、emission、Yuma Consensus 那几页。最有意思的发现是 2025 年 11 月他们把 emission 分配从价格驱动改成了流量驱动（Taoflow），原因是有人发现可以人为拉 token 价格来套奖励。他们自己承认了这个漏洞然后修掉了，这个让我觉得这个项目至少是认真在跑的。但 weight copying 问题（验证者互相抄评分）我还没想明白 Yuma Consensus 3 有没有真的解决。Render Network 相对好理解，本质就是闲置 GPU 接活，Web3 是外壳。让我意外的是节点接入居然还要填 Interest Form 申请，"去中心化"这个标签打了不少折扣。

晚上搞定了信息流关注清单，X 上关注了 13 个账号。进阶部分去 @GoPlusSecurity 看了 Echo Protocol 被黑的分析，攻击者凭空铸了 1000 eBTC 借走 86 万刀。看完之后对上午做的那个签还是不签工具更有感触了，用户层的安全意识和协议层的漏洞是两个战场，但都是真实的。

今天新记住三个词。Prompt Injection 上面说了。Yuma Consensus 我最开始以为是类似 PoW/PoS 的共识机制，后来才发现它解决的问题完全不同。不是"谁来记账"，是"谁的 AI 产出更好"。TEE是 CPU 里的加密黑箱，Phala 用来做 AI Agent 的安全执行，跑完能出可验证证明，但它和 ZK 证明的区别我还没搞清楚，下次要挖一下。

第一天正式开始，比想象中混乱很多，但还好没放弃。

上午在啃 Handbook 的 LLM 章节。说实话一开始看不太进去，全是定义。直到看到LLM 本质是猜下一个词这句话才突然开窍——之前觉得 AI 很玄乎，什么神经网络深度学习，原来核心逻辑就是在做超级自动补全。做了个小实验，同一个问题「ETH Gas 是什么」，一次直接问，一次加了角色、任务、格式要求，回答质量差了好几个档次。才意识到自己之前用 AI 都太随意了，prompt 不是在提问，是在给 AI 设定一个说话的框架。

下午跟着做交易解释器，Node.js + ethers.js，说实话大部分代码我看不懂，是 CC 帮我写的。但整个逻辑我是理解的：输入一个 tx hash，工具去 Sepolia 链上读原始数据，自动解码 ERC20 Transfer 事件，然后把数据喂给 DeepSeek 生成人话解释。跑通的那一刻有点小兴奋，第一次感觉自己真的读懂了一笔链上交易，不是通过钱包界面看余额，是直接读底层数据。做的过程里有一个细节让我印象很深：要把链上能验证的事实和模型在推断的东西分开。比如转了 155 个 USDT+是链上写着的，谁都能查。但这笔钱是用来支付服务费的是 AI 猜的，不一定对。以前用 AI 从来没想过这个边界，现在会开始区分了。

晚上连着看了两场活动，有点累，老师讲的好快全是干货，我这个小白真是有点云里雾里。Co-learning 那场聊到 Hermes 本地部署的优势，还有 AI 工具选型，信息量挺大。Web3 架构那场有一句话记下来了：安全源于设计，不是事后修补。Web2 出了漏洞可以推更新打补丁，Web3 合约一旦部署上链基本改不了，设计的时候就得想好所有情况。还有私钥那部分，听到私钥丢了真的就没了，没有客服没有找回，有点慌，会后悄悄检查了一下自己的钱包备份。

下午做交易解释器的时候顺手发现了一个问题——之前 Hermes 帮我提交的 proof 全不合格，一条条都是光秃秃的 repo URL，平台要求的格式完全没满足。当时有点崩，只能暂停手头的事逐条重交，对照要求一个个改。有点烦，但也是教训——以后提交前要自己看一眼，不能完全甩给 AI。
