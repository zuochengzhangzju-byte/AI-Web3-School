8 ## 一、什么是 Claude Code？

9

10 Claude Code 是 Anthropic 推出的\*\*终端 AI 编程助手\*\*，直接集成在命令行中。它可以：

11

12 - 理解并修改代码库

13 - 执行 Git 操作（commit、push、PR）

14 - 运行测试、调试代码

15 - 与 MCP 服务器集成，扩展能力（如链上交互、合约验证）

16

17 对于 Web3 开发者来说，Claude Code + MCP = **AI Agent 驱动的智能合约开发工作流**。

97 ## 四、配置文件体系

98

99 Claude Code 采用\*\*多层配置覆盖\*\*机制：

100

101 \`\`\`

102 ~/.claude/

103 ├── settings.json # 全局配置（用户级，不提交）

104 └── [CLAUDE.md](http://CLAUDE.md) # 全局指令

105

106 my-web3-project/

107 ├── .claude/

108 │ ├── settings.json # 项目共享配置（可提交 Git）

109 │ └── settings.local.json # 项目本地配置（自动 gitignore，不提交）

110 ├── [CLAUDE.md](http://CLAUDE.md) # 项目指令

111 └── .mcp.json # MCP 服务注册表

112 \`\`\`

113

114 ### 优先级（从高到低）

115

116 1. `.claude/settings.local.json` — 个人覆盖（密钥、实验性配置）

117 2. `.claude/settings.json` — 项目共享

118 3. `~/.claude/settings.json` — 全局用户

185 ## 六、Hooks（钩子）自动化

186

187 Hooks 可以在特定事件发生时自动执行命令，是实现\*\*自动化工作流\*\*的关键。

188

189 ### 6.1 支持的事件（部分）

190

191 | 事件 | 触发时机 |

192 |------|---------|

193 | `SessionStart` | 会话开始或恢复 |

194 | `UserPromptSubmit` | 用户提交消息后 |

195 | `PreToolUse` | 工具执行前（可阻止） |

196 | `PostToolUse` | 工具成功执行后 |

197 | `PostToolUseFailure` | 工具执行失败后 |

198 | `Stop` | Claude 完成响应 |

199

200 ### 6.2 实用示例

201

202 **编辑后自动格式化代码：**

203

204 \`\`\`json

205 {

206 "hooks": {

207 "PostToolUse": \[{

208 "matcher": "Edit|Write",

209 "hooks": \[{

210 "type": "command",

211 "command": "npx prettier --write \\"$CLAUDE\_PROJECT\_DIR/\*\*/\*.{ts,rs,js,json}\\"",

212 "timeout": 30

213 }\]

214 }\]

215 }

216 }

217 \`\`\`

218

219 **每次提交前运行测试：**

220

221 \`\`\`json

222 {

223 "hooks": {

224 "PreToolUse": \[{

225 "matcher": "Bash(git commit \*)",

226 "hooks": \[{

227 "type": "command",

228 "command": "cargo test 2>&1 || anchor test 2>&1",

229 "timeout": 120

230 }\]

231 }\]

232 }

233 }

234 \`\`\`

235

236 **桌面通知（Claude 需要输入时）：**

237

238 \`\`\`json

239 {

240 "hooks": {

241 "Notification": \[{

242 "hooks": \[{

243 "type": "command",

244 "command": "osascript -e 'display notification \\"Claude Code 需要你\\" with title \\"Claude Code\\"'"

245 }\]

246 }\]

247 }

248 }

249 \`\`\`

250

251 ### 6.3 5 种 Hook 类型

252

253 | 类型 | 说明 | 适用场景 |

254 |------|------|---------|

255 | `command` | 执行 Shell 命令 | 格式化、lint、测试 |

256 | `http` | 发送 HTTP 请求（v2.1.63+） | 通知 Webhook |

257 | `mcp_tool` | 调用 MCP 工具（v2.1.118+） | 链上验证 |

258 | `prompt` | 单轮 LLM 评估 | 代码审查 |

259 | `agent` | 多轮 Agent 验证（实验性） | 复杂检查 |

263 ## 七、MCP 服务器配置

264

265 MCP（Model Context Protocol）是 Claude Code 的\*\*能力扩展层\*\*，对于 Web3 开发尤为关键。

266

267 ### 7.1 基本配置

268

269 `.mcp.json`（项目根目录）：

270

271 \`\`\`json

272 {

273 "mcpServers": {

274 "solana-tools": {

275 "command": "npx",

276 "args": \["@solana/web3.js-mcp-server"\]

277 },

278 "contract-verifier": {

279 "command": "python",

280 "args": \["path/to/[verifier-server.py](http://verifier-server.py)"\],

281 "env": {

282 "EXPLORER\_API": "[https://api.devnet.solana.com](https://api.devnet.solana.com)"

283 }

284 }

285 }

286 }

287 \`\`\`

288

289 ### 7.2 Web3 开发推荐 MCP 服务

290

291 | MCP 服务 | 用途 |

292 |----------|------|

293 | `solana-mcp-server` | Solana 链上交互、账户查询 |

294 | `anchor-mcp-server` | Anchor 框架合约部署验证 |

295 | `ipfs-mcp-server` | IPFS 文件存储与检索 |

296 | `explorer-mcp-server` | 区块链浏览器数据查询 |

297

298 ### 7.3 MCP + Hooks 联动

299

300 在 Hook 中直接调用 MCP 工具：

301

302 \`\`\`json

303 {

304 "hooks": {

305 "PostToolUse": \[{

306 "matcher": "Write(\*.sol)",

307 "hooks": \[{

308 "type": "mcp\_tool",

309 "server": "solana-tools",

310 "tool": "verify\_contract"

311 }\]

312 }\]

313 }

314 }

315 \`\`\`

316

317 > 这样每次写完 Solidity/Rust 合约代码，自动触发链上验证。

428 ## 十、常见问题

429

430 ### Q1: 国内无法访问 Claude API？

431

432 使用阿里云百炼/百度千帆等国内服务商中转。参考第三节配置。

433

434 ### Q2: 权限弹窗太多？

435

436 设置 `"permissions": { "defaultMode": "auto" }` 自动放行安全操作，或用 `/fewer-permission-prompts` 命令优化。

437

438 ### Q3: Hooks 不生效？

439

440 - 检查 Hook 事件名是否\*\*区分大小写\*\*（如 `PostToolUse` 不是 `posttooluse`）

441 - Hook 默认超时 60 秒，长任务需设置 `timeout`

442 - 退出码 `2` 会阻止操作（PreToolUse 场景）

443

444 ### Q4: MCP 服务器连接失败？

445

446 - 确认 `command` 和 `args` 中的路径正确

447 - 检查环境变量是否通过 `env` 字段正确传递

448 - 运行 `claude doctor` 诊断

449

450 ### Q5: 如何在项目中共享配置？

451

452 将 `.claude/settings.jsonCLAUDE.md.mcp.json` 提交到 Git 仓库，队友 `git clone` 后开箱即用。
