# **AI Agent 入门：Hermes 从 0 到 1 学习笔记**

## **一、角色与目标**

-   **个人 Learning Agent 的定位**：AI × Web3 School 为学员提供了个人学习助理。Learning Agent 并不是替学员完成所有任务，而是帮助学员理解课程、规划每日任务并维护个人学习仓库，同时生成打卡草稿，并提醒学员同步到 WCB/打卡平台。
    
-   **固定入口**：日常使用时建议常用以下链接：Handbook（课程手册）、WCB 课程和 Learning 页面、WCB Agent API 文档、GitHub 官网及 CLI 等 。
    

## **二、初始化流程**

1.  **确认学员画像**：
    
    -   问题涉及学员的 AI 基础、Web3 基础、编程能力、目标方向、每日投入时间和输出语言等 。
        
    -   每轮提问控制在 2 – 3 个问题，收集信息后需复述给学员确认。
        
2.  **引导安装和登录 GitHub CLI**：
    
    -   学员需先注册或登录 GitHub，随后安装 GitHub CLI 并通过 `gh auth login` 完成授权 。
        
    -   重要提示：不要要求学员发送 Personal Access Token、密码或验证码 。
        
3.  **创建个人学习仓库**：
    
    -   推荐仓库名为 `ai-web3-school-cohort-0`，设置为 public，并将本地目录设在 `~/ai-web3-school-cohort-0` 。
        
    -   仓库描述建议为「Personal learning journal and proof‑of‑work for AI × Web3 School」。
        
    -   仓库默认公开，禁止上传任何隐私信息、API key、助记词或未公开的联系信息 。
        
    -   创建仓库后，若 Agent 对本地 repo 做了任何变更（创建、修改笔记或整理反馈），应在学员确认后执行 `git add .` → `git commit` → `git push`，并保持简短提交信息 。
        
4.  **初始化仓库结构**：
    
    -   建议在仓库中创建以下文件和目录：
        
        -   `README.md`：存放 AI × Web3 School 简介、Handbook 链接、WCB 课程链接和隐私提醒 。
            
        -   `profile.md`：记录个人信息与学习画像。
            
        -   `learning-plan.md`：制定学习计划。
            
        -   子目录：`daily/`（每日笔记）、`tasks/`（任务记录）、`experiments/`、`handbook-feedback/`、`hackathon/`、`submissions/`。
            
        -   模板目录：`templates/daily-note.md`、`templates/task-note.md`。
            

## **三、每日学习与打卡**

-   **提醒频次**：Agent 可以在早上和/或晚上发送提醒，具体频次根据学员选择 。
    
-   **每日流程**：
    
    1.  查看 WCB Learning 页面，确认当天的课程、任务、会议和打卡入口。
        
    2.  阅读 Handbook 中相关章节，生成 **今日最小路径**、**推荐路径** 和 **挑战路径**。
        
    3.  在 `daily/YYYY-MM-DD.md` 中撰写每日笔记。
        
    4.  生成打卡草稿，并返回 WCB / 打卡平台的链接供学员手动提交 。
        
    5.  学员提交后，将打卡链接或提交记录写回每日笔记。
        
-   **重要说明**：不承诺“一键同步”至原生平台；默认是生成打卡内容 + 返回打卡链接，由学员确认后手动提交 。
    

## **四、Handbook 反馈**

-   学员在学习中遇到的问题或建议（概念不清、资料过期、结构建议等）应归档在 `handbook-feedback/` 目录 。
    
-   每条反馈建议包含：对应 Handbook 页面链接、问题描述、改进建议和信息来源。
    

## **五、连接 WCB Agent API 与 Secret 管理**

-   若需调用 WCB Agent API，使用线上 Base URL：[https://web3career.build](https://web3career.build)；API 文档参照 [https://web3career.build/llms.txt](https://web3career.build/llms.txt) 。
    
-   Secret API Key 应存放在本地环境变量或 Hermes secrets（如 `WCB_AGENT_SECRET_API_KEY`），禁止写入 prompt、README 或公开仓库 。
    
-   任何写入型操作（提交任务、更新资料、创建记录）都需先展示即将提交的内容，并取得学员确认 。
    

## **六、最终输出与设计原则**

-   **最终输出**：完成初始化后，Learning Agent 不要生成冗长的报告，而是给出一份精简清单，包括：学员画像摘要、GitHub repo URL、本地目录、Handbook 链接、WCB Learning 链接、下一步行动、打卡链接、提醒设置（早/晚/未设置）和待办事项 。
    
-   **设计原则**：
    
    -   **轻量优先**：优先帮助学员即刻行动，而非一次性规划全部未来任务 。
        
    -   **人工确认**：涉及账号、repo、文件写入、打卡或 API 调用等操作必须征得学员确认 。
        
    -   **开源沉淀**：个人仓库不仅是笔记，更是证明学习成果的 workspace 。
        
    -   **隐私安全**：public repo 不应存放任何敏感信息 。
        
    -   **反馈闭环**：学习过程中的问题应回流至 Handbook feedback，实现知识的迭代改进 。
        
    -   **平台边界**：Agent 只承担辅助生成和提醒，正式提交以 WCB/打卡平台为准 。
        

## **七、课前准备和关联任务**

-   **课前准备**：根据课程安排，提前打开《AI × Web3 School Learning Agent 启动 Prompt》文件并阅读；同时准备一个 GitHub 仓库或本地学习目录，用于后续的学习笔记和 Proof‑of‑Work。
    
-   **关联任务**：按时参加实时课堂或观看回放，在实践中熟悉 Hermes Agent 的使用流程，将学习计划、Repo 管理、每日记录、任务提交与反馈环节串联起来。
