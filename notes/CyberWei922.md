# 2026-05-22
<!-- DAILY_CHECKIN_2026-05-22_START -->
最近在复盘大模型落地应用时，对 **Prompt、Workflow（工作流）** 和 **Agent（智能体）** 的边界、风险以及可调试性有了更深的体感。这三者不是简单的技术升级，而是三种完全不同的**人机协同范式**：

### 1\. Prompt（提示词）：人在前，模在后

-   **本质**：它是**单点式问答**。决策权、拼装权、上下文管理权全在人手里。模型只是一个高配的“文字渲染器”或“意图提取器”。
    
-   **风险与调试**：**基本在掌控之中。** 模型回答不好，人一眼就能看出来，改改提示词、加个 One-Shot（示例）就能立刻见效。它的失控局限在“这一轮对话里”。
    

### 2\. Workflow（工作流）：路径锁定，模型做节点

-   **本质**：它是**确定性的生产线**。人类提前用代码或低代码工具把路径死死卡住（例如：A节点提取关键词 -> B节点分类 -> C节点生成摘要）。模型在特定的格子里当“螺丝钉”。
    
-   **风险与调试**：**极高可预测性，局部可控。** 路径是死的，所以不会“迷路”。如果最终输出不对，你可以像传统软件测试一样，顺藤摸瓜去查到底是哪一个节点的提示词跑偏了，或者哪个接口报错了。它的风险在于“管道堵塞”，但非常好修。
    

### 3\. Agent（智能体）：模型在前的“黑盒探索”

-   **本质**：它是**目标导向的自主执行器**。人类只给一个模糊的目标（例如：“帮我分析竞品并出一份报告”），模型自己去拆解任务、自己决定什么时候查谷歌、什么时候调数据库、什么时候自己修正错误。
    
-   **风险与调试**：**失控风险急剧放大，调试变成“玄学”。**
    
    -   **死循环与幻觉放大**：Agent 在思考（Reasoning）和调用工具的环路中，一步错步步错，甚至可能陷入无限调用工具的死循环，或者在好几轮对话后彻底忘记最初的目标（状态管理失效）。
        
    -   **黑盒调试**：当 Agent 最终给出一个糟糕的结果时，你很难分清是因为目标定得不准、中间某一步工具调用参数错了、还是大模型突然“脑抽”没理解上下文。你没法像 Workflow 那样精准打断点，只能去对着长长的日志（Trace）猜它的心思。

# 2026-05-20
<!-- DAILY_CHECKIN_2026-05-20_START -->


-   LLM (大语言模型)：非确定性的高维“世界模型”，本质上是对人类知识高维压缩的概率机器，擅长语义推理与模式匹配而非精确计算。
    
-   Prompt (提示词)：高维参数空间的方向盘，通过高级语义对模型的概率场进行边界约束与上下文激活，从而拉平认知偏差。
    
-   Workflow (工作流)：确定性的刚性骨架，用代码锁死固定的业务逻辑与任务管线，是扼杀 LLM 幻觉、实现工业级落地的基石。
    
-   Tool Use (工具调用)：大模型的“行动义肢”，通过将自然语言意图转化为结构化 API 参数，让 AI 具备连接外部世界与实时数据的感知能力。
    
-   Agent (智能体)：具备自适应决策的数字生命，将大模型、记忆、规划与工具融为一体，能够自主拆解目标并进行闭环纠错。
    
-   AI Coding (AI 编程)：声明式的开发范式转移，将代码编写降维打击为自然语言的意图声明，使程序员的核心竞争力从“堆砌语法”升维至“架构审美与问题定义”。
