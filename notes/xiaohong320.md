# 2026-05-23
<!-- DAILY_CHECKIN_2026-05-23_START -->
这次课揭示了 AI 与 Web3 融合的本质，指出其核心绝非炒作发币，而是为下一代智能体（AI Agent）构建不可或缺的“经济基础设施”。当 AI 的演进从简单的“对话问答”迈向独立“执行任务”时，软件主体必然会遭遇身份授权、资产账户、支付清算以及合规审计等现实瓶颈，而区块链天然成为了完美的治理与微付款账本。

整个生态框架由浅入深分为五大核心环节： 在**底座层**，项目如 Bittensor、Render 和 Akash 通过去中心化市场组织算力与模型，实现“链下计算、链上激励”； 在**行动与认知层**，Coinbase AgentKit 等工具赋予 Agent 链上身份与数字钱包，而 Arkham 则负责将复杂的链上行为翻译为人类可懂的认知； 在**用户与机构防护层**，Blockaid 等安全助手在签名执行前进行交易模拟与风险拦截，将安全防线从单纯的界面美观提升至底层的语义理解。

总结而言，未来的行业边界由三大判断决定：Agent 必须拥有经济账户，所有项目都是在为该经济体补齐特定环节，而 Token 的真正价值在于让软件主体安全合规地融入结算系统。真正值得长期追踪的，必然是那些具备真实用户、工程接口和安全约束的基础设施级项目。

### **区块链交易：**

**Wallet（钱包 - 签名）**：用户在自己的钱包里用私钥对交易进行加密签名。这证明了“这笔钱是我要转的，不是别人盗用的”。**私钥永远不离开本地**。

-   **RPC / Node（节点 - 传播）**：钱包将签好的交易发送到区块链节点（RPC接口）。节点就像是网络中的传声筒，把交易广播给其他节点。
    
-   **Mempool（内存池 - 排队）**：这是交易的中转站。所有“还在路上、还没被确认”的交易都在这里排队等待被打包。
    
-   **Builder（构建者 - 排序）**：在以太坊的权益证明（PoS）机制中，构建者负责从Mempool里挑选出高手续费或特定的交易，并把它们排好序，组成一个候选的“块”。
    
-   **Validator（验证者 - 出块/证明）**：验证者（需要质押ETH的节点）被随机选中，来确认构建者排好的这个块是否合法。一旦同意，该块就被正式写入链上。
    
-   **Block（区块 - 可查询）**：交易最终被写进不可篡改的区块里。你可以在像 Etherscan 这样的区块链浏览器上公开查询到这笔交易的状态。
    

### **共识机制：**

这张图对比了区块链世界里两种最重要的“记账规则”——工作量证明（PoW）和权益证明（PoS）。

-   **PoW（BTC - 比特币）**
    
    -   **核心理念**：算力竞争。
        
    -   **比喻**：就像“解一道很难但答案很好验的数学题”。谁先算出来，谁就有资格记账。这种机制消耗大量电力，但极其安全。
        
-   **PoS（ETH - 以太坊）**
    
    -   **核心理念**：质押竞争。
        
    -   **运作方式**：验证者需要锁定（质押）一定数量的 ETH。系统会像摇号一样随机选取一名验证者来提议区块（Proposer），并随机选取其他验证者来证明它的正确性（Validator）。
        
    -   **惩罚机制**：如果你作恶或者严重掉线，你质押的币会被“罚没”（Slashing），以此激励大家诚实。
        

### **智能合约：**

-   **规则能看见（透明性）**：传统合同是写在纸上的，或者躺在机构数据库里的。而智能合约的代码、调用记录和执行历史，**任何人**都可以在链上公开检查。无法藏猫腻。
    
-   **少一点临时拍板（确定性）**：代码是“死”的。只要条件满足（比如时间到了、转账到了），代码就会**自动执行**。这排除了人为的干预和临时更改规则的空间（比如“最后关头改法律”）。
    
-   **信任对象变了（Trust Shift）**：
    
    -   传统模式是：相信某个具体的机构（银行、政府、中介）。
        
    -   Web3模式是：信任转变为了**“检查代码 + 检查共识机制 + 审计/治理流程”**。

这周是安装和使用Hermes Agent

没什么笔记，让DS老师帮我写了一份介绍：

Hermes Agent 是一个由美国 AI 研究实验室 Nous Research 开发的开源 AI 智能体框架，因其名称发音，在国内开发者社区中常被亲切地称为“爱马仕”。

它并非一个简单的聊天机器人，而是一个\*\*能够部署在服务器上自主运行、具备持久记忆并能不断自我进化的数字助理\*\*。它的核心目标是解决传统 AI 助手“用完即走”、“记不住事”和“无法积累经验”的痛点，旨在成为一个“与你一同成长的智能体”（the agent that grows with you）。

\### 🧠 核心能力：它不仅“记得”，还能“成长”

Hermes Agent 的核心竞争力体现在以下几个方面：

\- **🔁 自进化技能系统**：这是它最独特的功能。当完成一项复杂任务后，Hermes 会在后台自动复盘，将解决问题的有效路径、踩过的坑和成功经验，\*\*自动总结并沉淀为一个可复用的“技能”\*\*。下次遇到类似任务，它就能直接调用，因此会“越用越聪明”。

\- **💬 持久化记忆系统**：它通过管理 `MEMORY.md`（存储项目信息、技术决策）和 `USER.md`（存储用户偏好）两个核心文件，实现了跨会话的长期记忆。这意味着你昨天教它的编码规范，它今天依然会遵守，无需重复说明。

\- **🛠️ 强大的工具与集成能力**：Hermes 不只是会聊天，它能真正“动手干活”。它内置了 40 多种工具，可以执行终端命令、读写文件、进行浏览器自动化，并支持定时任务。同时，它能无缝接入 Telegram、微信等主流平台，让你随时随地给它派发任务。

\- **⚙️ 灵活的部署与模型选择**：它支持本地、Docker 或云服务器等多种部署方式，官方表示 5 美元/月的 VPS 就能流畅运行。在模型方面，它兼容 OpenAI、Anthropic、DeepSeek 及国内多款主流模型，并支持根据任务复杂度动态切换，以优化成本和效果。

\### ⚖️ 与 OpenClaw 的对比：两个明星项目的差异

在当前的开源 AI 智能体领域，Hermes Agent 和较早走红的 **OpenClaw（被社区称为“龙虾”）** 是经常被比较的两个明星项目。它们功能相似，但设计哲学有显著不同。

| 特性维度 | 🤖 Hermes Agent（爱马仕） | 🦞 OpenClaw（龙虾） |

| :--- | :--- | :--- |

| **核心哲学** | **自进化**，通过事后复盘将经验固化为“技能”来成长 | **生态与工具**，依赖庞大且成熟的预置技能库和社区生态 |

| **记忆与学习** | **主动进行**。系统会\*\*自动\*\*在后台回顾、总结并生成新技能，实现自我迭代 | **用户驱动**。虽也有记忆和技能功能，但更多需要用户\*\*主动创建或安装\*\*，本身不强调自动进化 |

| **成本与效率** | 采用“懒加载”策略，按需加载技能定义，在相同任务下\*\*Token 消耗显著更低\*\*，成本更友好 | 倾向于默认加载所有已安装技能，上下文开销大，导致 **Token 消耗较高** |

| **社区与热度** | **后起之秀**，凭借“自进化”理念增长迅猛，对中文社区支持友好 | **老牌强者**，生态最完善，用户基数庞大，在GitHub上拥有更高的提交频率和星标数 |

简单来说，OpenClaw像一个功能全面的“瑞士军刀”，而 Hermes Agent 则像一个能记住你习惯、并不断自我优化的“专属管家”。

\### 📈 部署与使用

得益于社区和云服务商的快速跟进，部署 Hermes Agent 已变得非常简便。例如，\*\*腾讯云轻量应用服务器（Lighthouse）\*\* 已上线其专属应用模板，可以实现“一键部署”。同样，在\*\*阿里云\*\*的计算巢中，也提供了相应的社区版部署方案。

对于开发者而言，基本的命令行安装也只需几行简单的指令，官方提供了完整的配置向导，门槛很低。

Web3 的交易与传统 Web2（如银行转账、支付宝）有着本质的区别。Web2 是中心化账本的数字加减；而 Web3 则是全球分布式的状态切换

web3交易步骤：

Construct： 用户在前端输入交易意图（比如：我要给 xxx 转账 1 个 ETH），此时交易还未生效，只是一段待签名的结构化数据。

Sign： 用户使用本地保管的私钥对构造好的交易结构体进行加密签名，证明交易确实由该资产的合法拥有者发起、交易内容在签名后无法被任何人篡改。

Broadcas： 钱包将带有签名的交易报文通过 RPC 接口发送到区块链节点。节点验证签名有效后，会将其扔进交易池（Mempool），全网的矿工/验证者（Validators）通过共识协议（PoW/PoS）将这笔交易打包进新的区块

钱包，钱包的本质是私钥管理工具与链上身份的鉴权通道。

根据账户控制的底层机制，钱包类型有：

EOA 钱包：一个私钥对应一个地址。优点是生态兼容性极好；缺点是只有单签机制，一旦私钥丢失或泄露，资产彻底清零。

合约钱包 ： 账户本身就是一个部署在链上的智能合约，资产由合约代码逻辑控制

AA 钱包把 EOA 和合约钱包结合，让用户不再需要死记硬背 12 个助记词。用户可以用邮箱、手机号或苹果指纹（Passkey）登录，甚至支持由第三方（如项目方）代付 Gas 费，极大地降低了 Web2 用户进入 Web3 的门槛。

多签：

MPC 多签： 利用密码学（如 gg18/20 协议）在链下将私钥生成为多个碎片（Shares），分发给不同的人。当需要转账时，大家在链下协同计算出一个签名。对于区块链来说，它只看到一个普通的单签交易。优点是隐私性高、Gas 费极便宜、跨链通用。

合约多签： 每个人拥有独立的 EOA 钱包，资产存在一个多签合约里。转账时，必须有 M/N（比如 3 个人中的 2 个人）在链上分别发起和确认签名，合约才会放款。优点是公开透明、链上可追溯；缺点是每次签名和执行都要消耗昂贵的链上 Gas 费。

Web3 安全的特征

安全绝对不是一个可以后期随时添加的插”，而是必须源于设计的核心属性。

不能“先跑起来再说“， 在 Web2 中可以迭代修 Bug，但在 Web3 中，逻辑漏洞和设计缺陷一旦上线，往往意味着资产的资产丧失与瞬间清零。

可解释性： 无法解释系统行为，就无法保证资产安全。
