# 2026-05-18
<!-- DAILY_CHECKIN_2026-05-18_START -->





| DAY1 |
| --Web3 支付系统对比 |
|   |
| Web2：用户 → 支付服务 → Banking Institution，各方通过 API Key/Secret Key 进行身份验证 |
|   |
| Web3：将 Banking Institution 替换为 Blockchain + 链上监听服务，逻辑本质相同 |
|   |
| Web3 通过算法保证数据不可篡改，合约逻辑公开透明，解决信任问题 |
|   |
| --钱包体系 |
|   |
| 核心：私钥 = 随机数，私钥泄露则资产完全失控且不可恢复 |
|   |
| 分类：EOA 钱包、合约钱包（Gnosis Safe）、AA 钱包、多签钱包、硬件钱包、隐私钱包、分层钱包 |
|   |
| 托管方式：自托管（最安全）、全托管（交易所）、混合托管（MPC 多签，双方各持一份私钥切片） |
|   |
| 主要风险：私钥泄露、签名欺骗（钓鱼攻击）、权限滥用 |
|   |
| --交易生命周期 |
|   |
| 构造交易 → 私钥签名 → 广播上链 → Gas Fee 计算（EIP-1559：base fee + tip fee + max fee） |
|   |
| 需等待多个区块确认，防止链重组（reorg）风险 |
|   |
| 签名前必须做交易模拟，签名后模拟无意义 |
|   |
| --服务端钱包安全方案 |
|   |
| 资金量拆分 + 多签方案，避免单点风险 |
|   |
| 流程：交易可视化 → 人工审核 → 交易模拟 → 可信执行环境（TEE）签名 |
|   |
| --AI 时代开发者能力模型 |
|   |
| AI 是放大器，但方案设计、架构判断、安全验收仍需人来主导 |
|   |
| 看不懂 AI 给的方案 = 被 AI 驾驭，应让 AI 教你直到看懂为止 |
|   |
| 核心能力：架构能力、Debug 能力、扎实的基础知识 |
