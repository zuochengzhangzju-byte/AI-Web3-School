# 2026-05-20

1.  今天补看了 5.18 关于 AI 时代 Web3 开发者架构能力的课程。最大的收获是：AI 并没有降低系统复杂度，只是放大了开发者的执行能力。方案设计、架构判断、代码验收和安全责任仍然在人手里。如果 AI 给出的方案我看不懂，说明不是可以直接采用，而是应该让 AI 反过来帮我补齐基础知识。
    
2.  Web3 支付系统可以用 Web2 支付系统来类比理解：区块链替代了银行或支付机构的位置，但链上监听、确认数、风险筛查、回调和清结算仍然需要工程系统承接。尤其是钱包、私钥、签名和交易模拟这些环节，不能只停留在概念层面，因为 Web3 的错误往往直接对应资产损失。
    
3.  5.19 的 Hermes Agent 课程让我更清楚了学习助手的实现路径。Agent 的价值不只是聊天，而是结合模型、工具、记忆、Skill 和消息网关，持续辅助学习计划、任务读取和复盘。但 API key、私钥、危险命令等都必须有明确边界，高风险动作不能交给 Agent 自动执行，需要人工确认。

# 2026-05-18

今天主要复盘了 AI x Web3 School 的开营仪式，对整个活动的结构和学习方式有了更清晰的认识。开营仪式本身不是一节技术课，更像是一次学习规则和方向说明：课程会围绕 AI 与 Web3 的交叉展开，从基础概念、工具实践、链上操作，到后续的项目构思和 Demo Day，逐步把大家带入一个真实的共学流程。

我比较有收获的一点是，这个训练营不是单纯给资料，而是通过任务、打卡、学分、助教答疑、共学时间和 Handbook，把学习过程变成持续输出。第一周的重点是建立共同语言：AI 侧要理解大模型、Agent、工具调用和 AI coding；Web3 侧要理解钱包、签名、交易、合约、测试网和区块浏览器。只有先把这些基础概念打通，后面讨论 AI 如何参与链上执行、支付、权限和协作才有意义。

开营仪式里提到的学习助手也让我印象比较深。它不是简单聊天，而是可以帮助读取课程任务、整理学习计划、提醒打卡、沉淀笔记和连接 GitHub。这让我意识到，AI 在学习中的价值不只是回答问题，而是帮助我把零散信息变成结构化行动。

接下来我会先跟着 Week 1 的节奏完成基础任务，重点关注 AI Agent 与 Web3 权限、支付、执行边界之间的关系。
