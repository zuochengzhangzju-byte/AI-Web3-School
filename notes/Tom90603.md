# 2026-05-25
<!-- DAILY_CHECKIN_2026-05-25_START -->




LLM 每次对话都是独立 session，Agent 的"记忆"本质上是外部存储 + 检索的工程问题，不是模型本身的能力。

1\. 记忆分层的思路——短期（当前对话上下文）→ 中期（session 间的关键事实）→ 长期（知识库/向量数据库），不同层用不同的存储和检索策略

2\. 检索质量决定 Agent 表现——不是存得越多越好，而是"在需要的时候能拿到最相关的内容"，RAG 的 chunk 策略和 embedding 模型选择很关键

3\. 记忆写回机制更重要——Agent 怎么判断"这个东西值得记住"？自动 vs 人工触发，这是个产品设计问题

可参考的项目/工具：

\- Mem0 / MemGPT / Letta——三个主流的 Agent 长期记忆方案

\- ChromaDB / Pinecone——向量数据库做记忆存储

\- Hermes Agent 本身的 memory 工具——其实就是一种轻量级的记忆写回机制

没做东西，但是了解了AI伴侣的实现？也算学习了AI相关的知识了罢，且非常贴合这个日子

输入感知（麦克风/摄像头）→ STT转文字 → LLM思考 → 向量库检索记忆 → TTS语音合成 → 视觉表现（Live2D/硬件动作）。具身智能要在此基础上再加一套**物理世界交互层**——空间感知、运动控制、仿真迁移、实时系统。

**可以实现的方案**

**方案1：全云端API + 开源前端**

-   大模型 + SillyTavern酒馆，普通电脑甚至手机就能跑
    

-   核心靠提示词工程锁死性格人设
    

-   优点：模型聪明、支持多模态；缺点：延迟高、有API账单
    

**方案2：全本地部署**

-   Ollama + 量化模型（Llama 3 8B / Qwen 2.5 7B），RTX 4060级别显卡可跑
    

-   模型量化把几十G的模型压到4-5G塞进显存
    

-   优点：延迟低、免费、无审查；缺点：显存限制跑不了大参数模型
    

**方案3：虚拟主播交互（类Neuro-sama）**

-   LLM + 流式TTS（Edge-TTS/ChatTTS）+ VTube Studio联动
    

-   Lip Sync嘴型同步根据音频频率实时匹配皮套嘴型
