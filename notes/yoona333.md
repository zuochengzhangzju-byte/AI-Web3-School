**Day 6｜Open Agentic Economy：从 ERC-8004 / ERC-8183 到 Builder Path**

今天参加了 AI x Web3 School 的线上活动：Open Agentic Economy: From ERC-8004 / ERC-8183 to Builder Path。

这场分享让我第一次比较系统地把 Agent、支付、身份、验证和 Builder 路径串在一起理解。之前我更多是在学习 AI Agent 的基础概念，比如 workflow、tool use、guardrails、tracing 和 human-in-the-loop；今天则进一步看到，当 Agent 不只是“帮人完成任务”，而是开始参与经济协作时，就必须考虑身份、信誉、验证、支付和结算的问题。

今天我重点理解了两个方向：

ERC-8004 更像是在回答：一个 Agent 如何被识别、被发现、被评价、被验证。

ERC-8183 更像是在回答：Agent 和 Agent 之间如何围绕一个任务建立付款、交付、验证和结算流程。

这让我意识到，AI × Web3 的关键不只是“让 Agent 调用链上工具”，而是要为 Agent 提供一套可信协作的基础设施。否则 Agent 可以执行任务，但别人未必知道它是谁、是否可靠、结果能否被验证、付款应该如何释放。

今天完成内容：

1\. 实时参加了 Open Agentic Economy 线上活动；

2\. 初步了解 ERC-8004 / ERC-8183 与 Agentic Economy 的关系；

3\. 思考了 Agent 身份、机器支付、任务验证和链上记录之间的连接；

4\. 进一步明确 AI × Web3 项目不能只关注 demo，还要关注协议标准、信任机制和安全边界；

5\. 记录了后续可以继续深入的问题：Agent 如何建立信誉？任务完成如何验证？机器支付如何避免滥用？Builder 可以从哪些小场景切入？

今天最大的收获是，我开始把 Agent 从个人效率工具理解为可以参与协作和交易的数字主体。如果一个 Agent 未来可以接任务、调用工具、完成交付、获得支付，那么链上身份、信誉记录、验证机制和支付协议就会变得非常重要。

**Day 5｜Co-learning 任务推进与 PoW 整理**

相比正式课程，Co-learning 更像是一个实时任务推进场。它不是单纯听分享，而是带着自己当前正在做的任务、repo、问题和 proof 过来，把模糊的问题拆成下一步可以执行的动作。

今天我重点做了几件事：

1\. 对照看板检查了当前任务完成情况；

2\. 继续整理 daily check-in 内容，保证每天的学习记录可以追踪；

3\. 检查 GitHub repo 中的 PoW 记录是否清晰；

4\. 思考每个任务应该提交什么 proof，例如公开笔记、repo 链接、文章内容或截图记录；

5\. 把前几天整理的 AI Agent、workflow、tool use、guardrails、tracing 等内容继续沉淀到学习记录中。

今天最大的感受是，PoW 不是为了“交作业”，而是为了让学习过程留下证据。以前我可能更关注“我有没有学完”，但现在会更关注“别人能不能通过我的 repo 或记录，看懂我学了什么、做了什么、下一步准备做什么”。
