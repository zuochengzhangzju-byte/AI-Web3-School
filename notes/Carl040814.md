### 模块 A — AI 基础：核心概念

**LLM 本质**：基于上下文进行概率生成，预测最合理的下一个 token 序列。

**四个关键概念**：

-   **Token**：最小读取单位，≈0.75 英文词 / 1 中文字
    
-   **Embedding**：文字转数学向量，表达语义相似度
    
-   **Transformer**：注意力机制让模型关注输入中重要的部分
    
-   **Hallucination（幻觉）**：自信编造不存在的信息——必须外部核实
    

**Prompt → Workflow → Agent 三层次**：

| 层次 | 决策者 | 风险 |
| --- | --- | --- |
| Prompt | 人在决策 | 低 |
| Workflow | 预定义流程 | 中 |
| Agent | 模型自主规划+工具调用 | 高 |

**Agent 核心组件**：Tool Calling、MCP、Skills、Tracing、Guardrails、Handoff、错误恢复、长期记忆。

### 模块 A — Prompt 深度

-   Instruction 四段式：角色→边界→异常处理→输出格式
    
-   Few-shot：放示例让模型模仿判断
    
-   Structured Output：JSON/schema 返回，用于后续检查
    
-   Prompt Injection：输入净化 + 权限分离 + 输出验证
    

### 模块 A — Agent 深度

-   核心分工：模型提方案→系统限空间→用户批边界
    
-   三大支柱：权限分级、可观测性、停止条件
    
-   Tool Use：schema、只读/写入、副作用、人工确认
    
-   Planning：拆步骤，暴露工具/读写/确认/重试
    
-   失败模式：循环、工具滥用、幻觉传递、权限蔓延
    

### 模块 C — AI × Web3 最小交叉实验

**实验**：在 Sepolia 测试网发送一笔 0.1 ETH 转账，用 AI Agent 逐层拆解。

**交易哈希**：`0xd7d486082ec118a9e9ec114e88a15abf4f00caea58b454ba43850c90d3b2f41f`

**交易全貌**：

```
FROM  0xb397...0139  ← 我的钱包
  │
  │  0.1 ETH（测试币）
  ↓
TO    0xb9db...8150  ← 接收方地址
```

**逐层对照**：

| 链上字段 | 数值 | 对应概念 |
| --- | --- | --- |
| Hash | 0xd7d4... | 密码学 — 交易唯一指纹 |
| r, s, v | 签名数据 | 密码学 — 私钥签名 + 公钥验证 |
| Nonce | 0 | 钱包 — 此地址第一笔交易 |
| Gas Limit | 31,500 | 钱包 — 愿意付的 gas 上限 |
| Gas Used | 21,000 | 钱包 — ETH 转账固定消耗 |
| Gas Price | ~2.9 Gwei | 钱包 — 每单位 gas 出价 |
| Chain ID | 11155111 | 网络 — Sepolia 测试网 |
| Type | 0x2 (EIP-1559) | 网络 — 动态费用机制 |
| Status | 0x1 (成功) | 合约 — 交易执行成功 |
| Block | #10886427 | 网络 — 被打包进这个区块 |

**交易生命周期**：

1.  MetaMask 点确认 → 本地签名（私钥不出设备）
    
2.  广播到 Sepolia mempool（待打包区）
    
3.  验证节点挑中 → 打包进区块 #10886427
    
4.  EVM 执行：from 减 0.1 ETH + gas，to 加 0.1 ETH
    
5.  状态写入链上，永久可查
    

**AI × Web3 交叉点**：如果 AI Agent 要代替人做这笔操作，至少需要：

-   额度限制（不能无限转账）
    
-   合约/地址白名单
    
-   交易模拟（提前看结果）
    
-   人工确认（高风险动作）
    
-   交易后审计记录
