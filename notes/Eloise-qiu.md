# 2026-05-19
<!-- DAILY_CHECKIN_2026-05-19_START -->
1.AI Coding 工具本质上都是基于已有大模型进行二次开发

2.主要工具：

聊天型 AI（ChatGPT、Gemini、Claude特点：对话式交互、通用问答、文本生成、代码辅助）

AI 编程助手：Cursor、Windsurf、Codeium、Claude Code特点：集成 IDE、辅助编程、自动补全、AI Debug、项目理解

终端型 Agent（CLI Agent）例如：Claude Code、OpenCode、Hermes Agent特点：、运行于 Terminal / CLI、可以调用本地工具、可以执行命令、更适合自动化开发流程

模型聚合平台：OpenRouter作用：聚合多个大模型、提供统一 API、方便切换模型、降低接入成本

3.Hermes Agent不仅是聊天机器人，更像：“AI 通用助手底座”可以连接多个大模型、调用本地工具、接入消息平台、管理 Memory、沉淀 Skills、自动执行任务

4.Hermes的核心能力:

Memory（记忆）：保存长期上下文,沉淀用户习惯,持久化记忆,并且不会无限膨胀，还会做整理与优化

Skills（技能）：固定流程化任务，学习计划生成，日报生成，新闻整理，量化分析，自动回复。当用户重复执行类似任务时，Hermes 会沉淀流程，自动调用已有 Skills。Skills 中会定义Agent 身份，行为边界，能力范围，执行流程

Tool Calling（工具调用）：读取本地文件，执行命令，调用 API，操作消息平台，联动 Telegram / 微信

5.让 AI 真正执行任务、调用工具、长期记忆、自动工作。

# 2026-05-18
<!-- DAILY_CHECKIN_2026-05-18_START -->

1.Web3和Web2并非完全独立，Web3项目中Web2能力占比也很高

2.在开发上，Web3不仅写智能合约，同时还需要大量外围的Web2工作支撑

3.AI运用的复杂度更看重人对其的使用，人作为主导设计方案和判断输出的质量，ai更多用于发散思维，如果人做为主体不能判断ai生成的内容是优质的还是劣质的，无异于是做了一次无效的AI使用

4.支付系统依赖于各方之间的API Key/Secret Key进行身份验证；在web3支付流程中，区块链在支付场景中定位与银行机构类比，区块链的信任机制通过共识算法保证链上数据的可信度，数据不可篡改，智能合约代码公开透明

5.钱包私钥不可以泄露，私钥丢失，资产无法恢复，找回的过程本质是和黑客抢交易，警惕钓鱼攻击，做好权限管理防止泄露

6.AI提升了开发效率，普通人能更快速的学习新技术，减少精力消耗；其次可以更深层次和更多维度的协助问题的梳理，但是AI无法进行系统复杂度判断，底层判断仍旧需要人来解决问题，出现的安全责任归属最终只能人进行承担
