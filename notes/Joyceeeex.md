# AI x Web3 分享笔记：为什么隐私对建设者至关重要

本次英文分享由 Web3 Privacy Now 核心成员 Peter 主讲，围绕新密码朋克（Neo-Cypherpunk）与 Web3 隐私建设展开。

Peter 介绍，Web3 Privacy Now 作为非营利智库，已联结全球 70 国参与者，致力于打破隐私领域圈层壁垒。他提出，Neo-Cypherpunk 是对 1993 年经典密码朋克的现代化升级，核心是将隐私视作公共品，以互助、开放、愉悦的社区精神，替代悲观对抗的旧理念，倡导人人可参与的隐私实践。

分享强调，隐私应从产品设计之初嵌入，而非事后补救。建设者可参考多元个人隐私工具栈，无需追求绝对隐私，从单一工具起步即可。同时指出，AI Agent 链上交互存在隐私空白，是重要创新方向。

针对大众隐私意识淡薄、监管监控等现实问题，Peter 认为隐私是程度选择，可通过实用方案规避风险。他批判加密行业过度金融化，呼吁回归以太坊 CROPS 价值（抗审查、开源、隐私、安全）。

最后，Peter 推荐免费开源的 Privacy Builder Pack 工具包，鼓励建设者立足隐私需求，打造更自由、普惠的 Web3 产品。

# 5.25 区块链隐私与应用实践学习笔记

## 一、区块链隐私必要性

链上公开透明可建立信任，但易暴露用户资产、交易行为，结合 IP 可定位真实身份；隐私保护是保障用户数字世界信息边界，而非隐匿不当行为。

## 二、RPC 与钱包隐私风险

RPC 是钱包与区块链中转站，中心化 RPC 易泄露地址、IP、交易数据，存在隐私泄露与服务宕机风险；可通过换服务商、去中心化 RPC、自建节点、隐私网络等优化。

## 三、理想钱包核心能力

具备交易匿名性、IP 隐私保护，支持自主验证链上数据，搭载轻客户端，减少对第三方依赖，兼顾隐私、安全与可用性。

## 四、隐私协议核心逻辑

采用资金池模型切断资金追踪，依托匿名集合提升隐私性；优先选用 UTXO 模型，资产独立、并发稳定，适配隐私交易与零知识证明。

## 五、合规平衡与现存挑战

需在隐私保护与资金合规间平衡，实现资金来源可验证；当前面临交易成本高、速度慢、数据同步难、用户体验差等问题。

## 六、应用实践方向

聚焦用户体验优化，落地隐私投票、匿名捐赠、AI+Web3 等应用；黑客松可从小切口切入，重协作学习与产品创新。

今天参加了「AI 下乡计划｜AI 在 Web3 的应用」会议。核心观点是：AI + Web3 的关键不在于“AI 发币”，而在于当 AI Agent 从“回答问题”走向“执行任务”时，Web3 为其提供身份、支付、结算、审计和风控等基础设施，从而形成面向 Agent 的经济系统。

内容主要包括五个方向：  
1）去中心化 AI 基础设施（如算力、存储、推理与服务网络）；  
2）AI Agent + 钱包，让 Agent 具备链上身份与支付执行能力；  
3）AI 驱动的链上分析工具，将复杂数据转化为可理解的判断；  
4）智能钱包与安全助手，在交易前进行风险识别；  
5）交易所与 DeFi 风控，通过模型与监控发现异常与风险。
