### **机器学习算法依托的问题场景**

机器学习在近30多年已发展为一门多领域交叉学科，涉及概率论、统计学、逼近论、凸分析、计算复杂性理论等多门学科。机器学习理论主要是设计和分析一些让计算机可以自动「学习」的算法。

**机器学习算法从数据中自动分析获得规律，并利用规律对未知数据进行预测**。机器学习理论关注可以实现的、行之有效的学习算法。很多推论问题属于无程序可循难度，所以部分的机器学习研究是开发容易处理的近似算法。

# 2026-05-25
<!-- DAILY_CHECKIN_2026-05-25_START -->


**模型**：在AI数据驱动的范畴内，模型指的是基于数据X做决策Y的假设函数，可以有不同的形态，计算型和规则型等。

**算法**：指学习模型的具体计算方法。统计学习基于训练数据集，根据学习策略，从假设空间中选择最优模型，最后需要考虑用什么样的计算方法求解最优模型。通常是一个最优化的问题。
<!-- DAILY_CHECKIN_2026-05-25_END -->

# 2026-05-23
<!-- DAILY_CHECKIN_2026-05-23_START -->



**监督学习**：从给定的训练数据集中学习出一个函数，当新的数据到来时，可以根据这个函数预测结果。监督学习的训练集要求是包括输入和输出，也可以说是特征和目标。训练集中的目标是由人标注的。常见的监督学习算法包括回归分析和统计分类。

-   更多监督学习的算法模型总结可以查看ShowMeAI的文章 [AI知识技能速查 | 机器学习-监督学习](https://www.showmeai.tech/article-detail/113)。
    

**无监督学习**：与监督学习相比，训练集没有人为标注的结果。常见的无监督学习算法有生成对抗网络（GAN）、聚类。

-   更多无监督学习的算法模型总结可以查看ShowMeAI的文章 [AI知识技能速查 | 机器学习-无监督学习](https://www.showmeai.tech/article-detail/114)。
    

**强化学习**：通过观察来学习做成如何的动作。每个动作都会对环境有所影响，学习对象根据观察到的周围环境的反馈来做出判断。

**如何防止过拟合呢**？一般的方法有Early Stopping、数据集扩增（Data Augmentation）、正则化、Dropout等。

-   **正则化**：指的是在目标函数后面添加一个正则化项，一般有L1正则化与L2正则化。L1正则是基于L1范数，即在目标函数后面加上参数的L1范数和项，即参数绝对值和与参数的积项。
    
-   **数据集扩增**：即需要得到更多的符合要求的数据，即和已有的数据是独立同分布的，或者近似独立同分布的。一般方法有：从数据源头采集更多数据、复制原有数据并加上随机噪声、重采样、根据当前数据集估计数据分布参数，使用该分布产生更多数据等。
    
-   **DropOut**：通过修改神经网络本身结构来实现的。

Harness Agent 和 OpenClaw 不在同一层级：Harness 是一种通用架构范式，而 OpenClaw 是基于该范式的具体产品实现。 一、核心定位 • Harness Agent ◦ 定义：一套通用AI Agent架构/方法论（Agent = Model + Harness） ◦ 本质：给大模型（LLM）套上“缰绳”，提供控制、执行、记忆、安全的底层框架 ◦ 形态：抽象框架，可落地为云/端/混合部署 ◦ 目标：解决AI Agent规模化、可靠落地问题 • OpenClaw ◦ 定义：开源、本地优先、开箱即用的AI智能体（数字员工） ◦ 本质：Harness思想的具体实现（个人助理/生活OS） ◦ 形态：桌面应用 + CLI工具，本地部署 ◦ 目标：让AI自主操作电脑、执行系统任务二、关键差异对比 • 抽象层级 ◦ Harness：方法论/架构范式（通用标准） ◦ OpenClaw：产品/实现（具体应用） • 模型支持 ◦ Harness：模型无关（适配任何LLM） ◦ OpenClaw：多模型兼容（Claude、GPT、本地模型） • 控制模式 ◦ Harness：人类主导、受控执行（Human-in-the-loop） ◦ OpenClaw：自主决策、最小人工干预• 技能系统 ◦ Harness：分层技能链、结构化调用 ◦ OpenClaw：ClawHub商店（1.3万+技能）、插件化 • 部署与数据 ◦ Harness：云/端/混合、企业级安全 ◦ OpenClaw：本地优先、数据私有（Markdown记忆） 三、一句话总结 • Harness = AI Agent的底层操作系统/工程范式 • OpenClaw = 基于Harness的、可直接使用的个人AI助手

最近不是这个叫 Hermes Agent 的东西特别火嘛，我也去摸了一下，最大的感受就是这玩意儿跟之前的 OpenClaw 虽然长得像，但内核完全是两码事。Hermes 的核心逻辑主打一个“自进化”——它干完活不会拍屁股走人，而是会回头复盘，把这次踩的坑和成功的经验自动提炼成 Skill 技能包存下来。这就意味着它真的越用越聪明，下次再干类似的活儿就不用重复交代了，不像 OpenClaw 那样 Skill 基本靠你手写，写多少它才会多少，不写就抓瞎。它的核心功能主要靠三个东西撑着：Memory（记你的偏好和环境）、Skill（存操作流程）、还有个 Nudge Engine（到点就提醒它该整理笔记了）。具体使用也挺简单的，我是用命令行一键装的，或者用 Docker 拉镜像也行，装完在 `.env` 里配一下模型 key 就能跑起来，它还能接 Telegram、微信这些入口当个常驻助理用。总之这玩意儿现在 GitHub 星标涨得飞快，都说它是“养成系”的 Agent，确实有点东西。
