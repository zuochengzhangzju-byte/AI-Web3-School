这个玩意是我2月在做独立站运营时候的灵感，市场的确存在这种需求，但信誉评分基于这种机制的产品商业化似乎存在困难，去年推特的ethos（好像是这个名字）活了两个月也见不到了，需要想其他方案

提交说明：信任交易 (TrustTrade-Demo)

仓库地址：[https://github.com/yuriswj12-bit/TrustTrade-Demo](https://github.com/yuriswj12-bit/TrustTrade-Demo)

1\. 解决什么学习问题

帮助学习「跨境贸易中的公开信誉评分系统」如何通过成功交易者留下互评辅助降低信任成本。普通人很难理解公开评价 + AI 分析如何实际运作，这个 Demo 让概念可交互、可体验。

2\. 用户如何交互

\- 在网页中查看卖家列表和实时评分

\- 提交公开评价（选择卖家 + 打分 + 写内容）

\- 通过 AI 助手用自然语言提问任意卖家评价情况

3\. 输入输出示例

输入：「天津服装贸易有限公司的评价怎么样」

输出：返回该卖家当前评分、所有公开评价列表，并给出「评分良好，可考虑合作」的 AI 建议。

4\. AI 生成与人工修改部分

\- AI 辅助：情感分析、关键词提取、自然语言查询回答、生成推荐标签

\- 人工部分：用户自己给出真实评分（1-100），AI 不会修改或生成评分

5\. 限制和下一步改进

当前 Demo 为纯前端模拟。是一个非常低级的demo，受限于现在个人能力无法做到满意，未来将引入区块链技术，使用 Hardhat + Solidity 智能合约存储评价与评分，结合 Circom + Groth16 零知识证明，实现隐私保护的信誉验证（例如在不暴露具体评价内容的情况下证明评分真实性）。

关键概念解释  
交易哈希：这笔交易在区块链上的唯一 ID。任何人输入这个哈希都能在区块浏览器查到完整详情，包括从哪里来、到哪里去、金额、Gas、手续费、状态、时间、所在区块。它是交易被打包进区块后自动生成的，无法伪造。  
  
区块浏览器显示内容：  
\- 交易状态：顶部绿色 Success 标签  
\- Gas：Transaction Fee（总手续费）和 Gas Price（每单位 Gas 价格，单位 Gwei）  
\- 区块高度：Block 字段显示具体区块号，旁边有确认数  
\- 时间：Timestamp 字段显示 UTC 时间和相对时间  
  
其他属性：  
\- 交易类型 2 (EIP-1559)：支持动态 base fee + priority fee  
\- 随机数 0：发送地址的第一笔交易  
\- 区块位置 41：这笔交易在区块中的排序位置

## 今日目标

围绕 Agent 交易产品做一次系统调研，重点看 EVM 和 SVM 两条链上目前有哪些可落地的 Agent 交易工具，以及这些工具对流动性交易者的实际价值。

## 已完成

今天我把主要精力放在了 Agent 交易产品的市场扫描上，目标是搞清楚目前哪些项目已经能跑通、哪些还停留在概念阶段。

Solana（SVM）侧我重点看了四个产品：

-   Pump.fun AI Skills（NexisSis）：核心是让自然语言直接控制 pump.fun 全流程。典型 prompt 包括「Snipe 这个 launch」「Copytrade 钱包 X」「自动 TP 50% / SL 30%」。这是我目前看到最接近实用的 meme 交易 agent 入口。
    
-   Warden Protocol：已经完成 100 万+ agent 触发的 swap，支持意图驱动的自主执行，链支持跨 Solana、Base、Ethereum。
    
-   Agentic Copy Traders（Allium Labs）：自动扫描高 PnL 聪明钱并镜像，强调过滤「类人类行为」以避开其他 bot。
    
-   Shooter Terminal / Snipoo：实时 pump.fun 扫描 + 自动 TP/SL + rug 检测。
    

Base（EVM）侧目前更偏基础设施和透明交易：

-   $BNKR：定位 agent 金融基础设施，agent 发 token 后交易手续费反哺 AI 智能、钱包、LLM gateway，已经有真实 Uniswap 成交量。
    
-   0xCoral：透明自主交易 agent，公开晒交易、推理过程和盈亏，builder 有 EF/NFT 背景。
    
-   $CLAWD：在 [base.org/agents](http://base.org/agents) 上架，同样有 EF 背景关联。
    

## 我的思考

从一个流动性交易者的角度看，Solana 侧的产品明显更贴近我的日常交易场景。Pump.fun + 自然语言 agent 几乎可以直接嵌入我的 meme 狙击 workflow，而 Warden 的跨链 swap 能力也解决了目前我手动切换链的痛点。

Base 侧虽然也有几个项目，但更多是「agent 经济」的实验。$BNKR 的费率反哺模型很有意思，不过离我真正能拿来 copytrade 还有距离。0xCoral 的透明晒单风格倒是值得学习，能减少黑箱风险。

所有项目都在反复强调 guardrails（护栏）的重要性。这点我非常认同。Agent 一旦失控，滑点、rug、极端行情下的错误决策都会放大损失。未来如果我自己要做 agent 交易工具，护栏设计必须是第一优先级。

这次调研也让我更清楚：目前市场还在「agent 能做什么」的验证阶段，真正稳定跑通的交易闭环还不多。这对我的交易策略是利好——窗口期还在，普通人还有机会找到属于自己的 agent 组合。

## 打卡要点

今天把 EVM/SVM 两条链上的主流 Agent 交易产品做了一次完整扫描。Solana 实用性明显领先，Base 更偏向基础设施和透明交易。Guardrails 是所有项目共同强调的核心风险控制点。

但考虑到链上流动性的问题，SOL的交易承载能力更强
